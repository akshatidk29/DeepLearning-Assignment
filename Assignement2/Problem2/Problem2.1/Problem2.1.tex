\section{Introduction}

This report investigates the robustness of two different neural network architectures when tested on spatially transformed images. We compare a Fully Connected Neural Network (FCNN) and a Convolutional Neural Network (CNN) on their ability to maintain accuracy when input images undergo spatial shifts.

\subsection{Objective}

The objective of this experiment is to understand how spatial transformations affect different network architectures and determine which design is more robust to position changes in input data.

\section{Models}

\subsection{Model A: Fully Connected Neural Network (FCNN)}

The FCNN architecture consists of:
\begin{itemize}
    \item Input layer: 784 neurons (28×28 flattened image)
    \item Hidden layer 1: 128 neurons with ReLU activation
    \item Hidden layer 2: 128 neurons with ReLU activation
    \item Hidden layer 3: 128 neurons with ReLU activation
    \item Output layer: 10 neurons (digit classes 0-9)
    \item Total parameters: 109,386
\end{itemize}

\subsection{Model B: Convolutional Neural Network (CNN)}

The CNN architecture consists of:
\begin{itemize}
    \item Convolutional Layer 1: 32 filters with 3×3 kernel, padding=1, ReLU activation
    \item Max Pooling 1: 2×2 kernel, stride=2
    \item Convolutional Layer 2: 64 filters with 3×3 kernel, padding=1, ReLU activation
    \item Max Pooling 2: 2×2 kernel, stride=2
    \item Fully Connected Layer 1: 128 neurons with ReLU activation
    \item Output layer: 10 neurons (digit classes 0-9)
    \item Total parameters: 409,098
\end{itemize}

\section{Training Parameters}

Both models were trained using the following configuration:
\begin{itemize}
    \item Dataset: MNIST handwritten digits
    \item Training samples: 60,000 images
    \item Test samples: 10,000 images
    \item Image size: 28×28 pixels (grayscale)
    \item Optimizer: Adam
    \item Learning rate: 0.001
    \item Batch size: 128
    \item Number of epochs: 10
    \item Loss function: Cross-entropy loss
\end{itemize}

The robustness test was conducted by creating a "Shifted MNIST" test set, where every image from the original test set was translated 4 pixels to the right.

\subsection{Key Observations}

\begin{itemize}
    \item Both models achieve high accuracy on the original test set (FCNN: 97.3\%, CNN: 99.0\%)
    \item FCNN experiences a severe accuracy drop of 62.04\% on shifted images
    \item CNN maintains much better robustness with only a 26.46\% accuracy drop
    \item CNN is approximately 2.3 times more robust than FCNN to spatial shifts
\end{itemize}

\section{Thoughtful Analysis}

\subsection{Why CNN Maintains Higher Accuracy Despite the Shift}

The large difference in robustness between the two models is mainly because of the \textbf{weight sharing property} used in convolution.

\vspace{0.5em}

\textbf{FCNN - Position-Specific Learning:}

In an FCNN, each neuron learns separate weights for every pixel in the image. This means it learns features based on exact positions. For example, if it learns a pattern at one location, it will only recognize it there. If the image shifts slightly, the pattern moves to a new position, and the neuron may fail to detect it. This makes FCNNs sensitive to small shifts.

\vspace{0.5em}

\textbf{CNN - Weight Sharing and Translation Equivariance:}

CNNs use the same filter across the whole image instead of different weights for each position. The filter slides over the image and looks for the same pattern everywhere. Because of this, even if the image shifts, the model can still detect the feature in its new position.

This property is called \textbf{translation equivariance}. It means that when the input shifts, the detected features also shift but are still recognized. So a filter trained to detect something like an edge can find it anywhere in the image, not just in one fixed spot.
